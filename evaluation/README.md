# Evaluation Module

## Dataset format
The evaluation suite is stored in CSV files under this folder.

- questions.csv contains the evaluation prompts with columns: id, department, difficulty, question, expected_behavior.
- ground_truth.csv contains the expected answer, document reference, section, and expected behavior for each prompt.
- results.csv is generated by the evaluator and stores the model answer plus a set of evaluation metrics.

## File descriptions
- questions.csv: evaluation prompts for the RAG system.
- ground_truth.csv: canonical answers and source metadata used for scoring.
- results.csv: runtime output with answer quality and grounding metrics.
- evaluate.py: script that reads the evaluation set, calls the backend /ask endpoint, and writes results.
- config.json: backend connection settings and evaluation defaults.
- metrics.py: heuristic scoring functions for retrieval, grounding, faithfulness, out-of-scope handling, and multi-hop success.

## How to run evaluate.py
From the repository root, run:

```bash
python evaluation/evaluate.py --backend-url http://127.0.0.1:8000
```

Use --dry-run to generate placeholder results without contacting the backend:

```bash
python evaluation/evaluate.py --dry-run
```

## Evaluation workflow
1. Start the backend service.
2. Ensure the /ask endpoint is reachable.
3. Run the evaluator to populate results.csv.
4. Review retrieval, groundedness, and out-of-scope metrics.
5. Update the report files in the reports directory with the first evaluation run results.

## Notes on scoring
The current scoring in metrics.py is a deterministic heuristic baseline. It is suitable for regression testing and CI workflows, and it can be replaced with embedding-based or LLM-as-judge evaluation when stronger grading is needed.
